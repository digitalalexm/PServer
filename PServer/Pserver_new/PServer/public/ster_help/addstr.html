-addstr
template: ster?clnt=<name|pass>&com=addstr&str=<str>[&<rule>=pattern1...&<rule>=patternN]
          Order of query params is not important. Stereotype
          names must not contain special characters '*'and must not be an empty string.
descript: A new stereotype based on the rules is inserted in the DB. The "rule" parameter is 
          a conjunction of logical operations on the demographic attributes. The form of the
          pattern is attr_name1$operator1$val1$operator2$val2;...attr_nameN[operatorK]valM...	 
	  A stereotype can have many rules. A user will be assign to the stereotype if his/her attributes 
	  satisfy one of the rules. 
          The operator for equality comparison is ':', for greater than >, greater equals to >:
          lesser or equals to <:, lesser than <, not equal <>. We use the : and not the = symbol 
	  because of the url character restrictions. 
	  If there is an existing stereotype with the same name an error will be returned.	  
          Note that if no rule parameter exists then pserver will create an empty stereotype and the
          users that will belong to this stereotype must . If a rule parameter exists pserver will 
          find all the users that satisfies the rule and will create the stereotype profile. 
example : ster?clnt=name|pass&com=addstr&str=it_expert&rule=computer_science_degree$:$yes;work_years$>:5
          ster?clnt=name|pass&com=addstr&str=teenager&rule=age$>:$13;age$<:$18
          ster?clnt=name|pass&com=addstr&str=notTeenager&rule=age$<:$13$>:$18
returns : 200 OK, 401 (fail, request error), 501 (fail, server error)
200 OK  : no response body exists.
